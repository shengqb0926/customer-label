# 推荐结果管理页面 - 前端测试指南

## 🎯 测试目标
在浏览器中验证推荐结果管理页面的所有功能

## 📋 测试步骤

### 第一步：访问页面

1. **打开浏览器**，访问：`http://localhost:5176/recommendations`
2. **预期结果**: 
   - ✅ 页面正常加载
   - ✅ 显示推荐列表表格
   - ✅ 顶部显示统计卡片（总数、待处理、已接受、已拒绝）
   - ✅ 侧边栏菜单"推荐结果"高亮

---

### 第二步：基础展示验证

#### 2.1 表格数据渲染

- [ ] 列头包括：客户 ID、标签名称、标签类别、置信度、来源、状态、推荐时间、操作

### 第四步：操作功能测试

#### 4.1 接受推荐
1. 找到一条"待处理"状态的推荐
2. 点击"接受"按钮
3. **预期**:
   - [ ] 弹出确认对话框
   - [ ] 确认后显示成功提示
   - [ ] 该行状态变为"已接受"（绿色 Tag）
   - [ ] 统计数据更新（待处理 -1，已接受 +1）

**验证命令**:
```bash
curl -s "http://localhost:3000/api/v1/recommendations/1/accept" \
-X POST \
-H "Content-Type: application/json" \
-H "Authorization: Bearer <YOUR_TOKEN>" \
-d '{"feedbackReason":"测试接受"}'
```

#### 4.2 拒绝推荐
1. 找到一条"待处理"状态的推荐
2. 点击"拒绝"按钮
3. **预期**:
   - [ ] 弹出确认对话框
   - [ ] 输入拒绝原因
   - [ ] 确认后显示成功提示
   - [ ] 该行状态保持"待处理"但标记为已拒绝
   - [ ] 统计数据更新

**验证命令**:
```bash

-X POST \
-H "Content-Type: application/json" \
-H "Authorization: Bearer <YOUR_TOKEN>" \
```

## 🔍 问题排查清单

### 如果页面无法加载：
1. 检查浏览器 Console 是否有错误
2. 验证后端服务是否运行：`curl http://localhost:3000/api/v1/health`
3. 检查前端编译日志：查看终端输出

### 如果筛选不生效：
1. 检查前端是否正确传递查询参数
2. 使用 curl 直接测试后端 API 验证筛选功能
3. 查看 Network 面板的请求参数

### 如果操作失败：
1. 检查 Token 是否过期（Local Storage 中的 `access_token`）
2. 查看 Network 面板的响应状态码
3. 检查后端日志中的错误信息
