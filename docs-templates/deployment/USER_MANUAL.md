### 2.3 查看客户详情

**场景**: 全面了解某个客户的背景信息和推荐历史

**操作步骤**:
```
1. 在客户列表中找到目标客户
2. 点击"查看详情"按钮
3. 查看完整信息:
   - 基本信息（姓名/邮箱/电话/年龄）
   - 资产状况（总资产/年收入/信用评分）
   - RFM 分析（R/F/M 得分及五分位排名）
   - 已有标签列表
   - 推荐历史记录（时间/引擎/结果）
```

**RFM 分析解读**:
```
R 值（Recency）: 最近消费时间得分（1-5 分）
  5 分 = 最近 7 天有交易
  1 分 = 超过 180 天无交易

F 值（Frequency）: 消费频率得分（1-5 分）
  5 分 = 年交易>50 次
  1 分 = 年交易<5 次

M 值（Monetary）: 消费金额得分（1-5 分）
  5 分 = 总资产>1000 万
  1 分 = 总资产<50 万

客户分群:
- RFM 总分 13-15 分：钻石客户（重点维护）
- RFM 总分 10-12 分：黄金客户（积极跟进）
- RFM 总分 6-9 分：潜力客户（挖掘需求）
- RFM 总分 3-5 分：一般客户（保持联系）
```

### 2.7 批量导入客户

2. 填写客户数据:
   - 按列头要求填写

   - 点击"导入"按钮
   - 选择填写好的 Excel 文件

## 🎯 三、推荐引擎

### 3.1 手动触发推荐引擎

**场景**: 为特定客户生成智能标签推荐

**重要原则**:
- ⚠️ **严禁自动触发**: 系统不会自动执行推荐
- ✅ **手动操作**: 必须由用户主动点击按钮
- ✅ **明确反馈**: 执行后会显示生成数量

**操作步骤**:
```
1. 在客户列表中找到目标客户
2. 根据需求选择引擎按钮:
   
   【规则引擎】- 适合明确业务规则场景
   图标：⚡
   说明：基于预设的业务规则匹配
   适用：高价值客户识别、VIP 升级等
   
   【聚合引擎】- 适合发现潜在客户群
   图标：🔍
   说明：K-Means 聚类分析
   适用：客户分群、相似客户发现
   
   【关联引擎】- 适合挖掘关联关系
   图标：🔗
   说明：Apriori 关联规则挖掘
   适用：交叉销售、产品推荐
   
   【全部引擎】- 综合推荐（推荐）
   图标：✨
   说明：并行执行三个引擎并融合结果
   适用：全面客户分析

3. 点击选择的引擎按钮
4. 系统显示 Loading 提示（通常 2-5 秒）
5. 执行完成后显示成功消息:
   "推荐生成成功！共生成 8 条推荐，已推送到推荐列表"
6. 点击"查看推荐"可跳转到推荐列表
```

**引擎选择建议**:
| 场景 | 推荐引擎 | 理由 |
|------|---------|------|
| **快速评估** | 规则引擎 | 执行快（<1s），规则透明 |
| **深度分析** | 全部引擎 | 多维度分析，准确率高 |
| **客户分群** | 聚合引擎 | 自动发现特征群体 |
| **交叉销售** | 关联引擎 | 挖掘产品关联关系 |

| `>` | 大于 | totalAssets > 5000000 |
| `>=` | 大于等于 | annualIncome >= 1000000 |
| `<` | 小于 | age < 30 |
| `<=` | 小于等于 | rfmTotal <= 5 |

### 5.2 聚类配置

**路径**: 配置管理 → 聚类配置

**调整 K 值**:
```
1. 点击"编辑"按钮
2. 修改参数:
   - K 值：簇数量（2-20）
   - 最大迭代次数：10-200
   - 初始化方法：random / kmeans++
   - 距离度量：euclidean / manhattan / cosine
3. 点击"保存"
4. 下次聚类时使用新参数
```

**K 值选择建议**:
- K=3~5: 粗粒度分群（适合小型数据集）
- K=5~10: 中等粒度（推荐，适合大多数场景）
- K=10~20: 细粒度分群（适合大型数据集）

**效果评估**:
- 轮廓系数（Silhouette Score）: -1 到 1，越接近 1 越好
- 簇内误差平方和（SSE）: 越小越好

---

### 5.3 关联配置

**路径**: 配置管理 → 关联配置

**调整阈值**:
```
1. 点击"编辑"按钮
2. 修改参数:
   - 最小支持度：0.01-1.0（默认 0.1）
   - 最小置信度：0.01-1.0（默认 0.6）
   - 最小提升度：1.0-10.0（默认 2.0）
3. 点击"保存"
```

**参数说明**:
- **支持度**: 规则在所有交易中出现的频率
  - 支持度越高，规则越普遍
  - 过低会过滤掉稀有但有价值的规则
  
- **置信度**: A 发生时 B 也发生的概率
  - 置信度越高，规则越可靠
  - 过高可能过滤掉有价值的弱关联
  
- **提升度**: B 在 A 发生时的概率与 B 总体概率的比值
  - 提升度>1 表示正相关
  - 提升度<1 表示负相关
  - 提升度=1 表示独立

**调优建议**:
```
场景 1: 规则太多，难以筛选
→ 提高最小支持度（0.1 → 0.2）
→ 提高最小置信度（0.6 → 0.8）

场景 2: 规则太少，没有推荐
→ 降低最小支持度（0.1 → 0.05）
→ 降低最小置信度（0.6 → 0.4）
```

## ❓ 六、常见问题 FAQ

### Q1: 推荐引擎执行失败怎么办？

**A**: 按以下步骤排查:
1. 检查网络连接是否正常
2. 查看浏览器控制台是否有报错
3. 确认客户数据是否完整（至少要有基础信息）
4. 联系管理员查看后端日志
5. 尝试刷新页面后重新执行

---

### Q2: 为什么我的推荐接受率很低？

**A**: 可能原因:
1. **置信度阈值过低**: 建议只接受>0.7 的推荐
2. **规则配置不合理**: 检查规则条件是否过于宽松
3. **数据质量问题**: 客户信息不完整或不准确
4. **算法参数不当**: 调整聚类 K 值或关联阈值

**改进建议**:
- 定期复盘拒绝的推荐，找出共同特征
- 根据业务反馈调整规则和参数
- 补充完善客户数据
