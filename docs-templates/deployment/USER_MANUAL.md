### 2.4 手动触发推荐引擎

**场景**: 为客户生成新的推荐标签

**重要提示**: 
- ⚠️ **严禁自动触发**：查看客户详情时不会自动执行推荐
- ✅ **手动触发原则**：必须由用户主动点击按钮执行
- 📊 **反馈机制**：执行后明确提示生成结果数量

**操作步骤**:
```
1. 在客户列表中找到目标客户
2. 点击操作列的【🎯 推荐】按钮
3. 选择推荐引擎模式：
   ○ 规则引擎（基于预设规则）
   ○ 聚类引擎（基于相似客户）
   ○ 关联引擎（基于关联规则）
   ◎ 全部引擎（综合推荐）
4. 点击"执行推荐"
5. 等待执行完成（通常 < 3 秒）
6. 查看结果提示："成功生成 3 条推荐"
7. 系统自动跳转到推荐管理页面查看明细
```

**引擎选择建议**:
| 引擎 | 适用场景 | 优势 | 执行时间 |
|------|---------|------|---------|
| **规则引擎** | 明确业务规则 | 精准、可解释 | < 1 秒 |
| **聚类引擎** | 发现潜在特征 | 智能化、创新 | 1-3 秒 |
| **关联引擎** | 挖掘关联关系 | 发现隐性规律 | 2-5 秒 |
| **全部引擎** | 全面分析 | 综合最优 | 3-8 秒 |

**API 调用详情**:
```
请求：POST /api/v1/recommendations/generate/:customerId
参数：mode = rule | cluster | association | all
响应：{
  "success": true,
  "data": {
    "generated": 3,
    "recommendations": [...]
  }
}
```

```

   ○ 标签不准确
   ○ 客户不符合条件

```

- 🔴 推荐状态变为 REJECTED
- 📝 记录拒绝原因和操作人
- 🚫 相同规则不会再次推荐该标签（除非重新训练）

## 📊 四、统计分析

### 4.1 RFM 分析

**路径**: 统计分析 → RFM 分析

**RFM 模型说明**:
```
R (Recency) - 最近一次消费时间
  5 分：7 天内
  4 分：30 天内
  3 分：90 天内
  2 分：180 天内
  1 分：超过 180 天

F (Frequency) - 消费频率
  5 分：年交易>50 次
  4 分：年交易 30-50 次
  3 分：年交易 15-29 次
  2 分：年交易 5-14 次
  1 分：年交易<5 次

M (Monetary) - 消费金额
  5 分：总资产>1000 万
  4 分：总资产 500-1000 万
  3 分：总资产 200-500 万
  2 分：总资产 50-200 万
  1 分：总资产<50 万
```

**客群分类**:
```
重要价值客户 (13-15 分): 重点维护，提供 VIP 服务
重要发展客户 (10-12 分): 积极跟进，挖掘潜力
重要保持客户 (7-9 分): 定期联系，防止流失
一般客户 (3-6 分): 保持接触，等待时机
```

### 6.2 推荐问题

**Q: 为什么没有生成推荐？**
A: 可能原因：
1. 客户数据不完整（缺少资产/交易信息）
2. 推荐引擎配置未启用
3. 不满足任何推荐规则

**Q: 推荐接受率很低怎么办？**
A: 建议：
1. 检查推荐规则配置
2. 调整规则阈值
3. 查看拒绝原因统计
4. 优化引擎算法

### 6.3 性能问题

**Q: 系统响应慢？**
A: 可能原因：
1. 网络延迟
2. 浏览器缓存过多
3. 同时在线用户过多

**解决方案**:
- 清理浏览器缓存
- 刷新页面重试
- 避开高峰时段
