---

### **2. 测试功能流程**

#### **测试 1：创建聚类配置**
1. 访问 `http://localhost:5176/clustering-configs`
2. 点击 **"新建配置"** 按钮
3. 填写表单：
   - 配置名称：`客户分群配置 V1`
   - 描述：`基于 K-Means 的客户分群`
   - 算法类型：选择 `K-Means`
   - K 值：`5`
   - 最大迭代次数：`100`
   - 是否激活：开启
4. 点击 **"确定"** 保存
5. 验证：列表中显示新配置

#### **测试 2：运行聚类任务**
1. 在配置列表中找到刚创建的配置
2. 确保状态为 **"活跃"**
3. 点击 **"运行"** 按钮
4. 等待执行完成
5. 查看成功提示消息

#### **测试 3：查看引擎执行记录**
1. 访问 `http://localhost:5176/engine-monitor`
2. 查看顶部的统计卡片
3. 在表格中查看执行历史
4. 尝试使用筛选条件：
   - 选择引擎类型（规则/聚合/关联）
   - 选择日期范围
   - 搜索客户名称

#### **测试 4：管理配置状态**
1. 在聚类配置页面
2. 找到任意配置
3. 点击状态开关（开→关）
4. 验证：状态变为 **"停用"**
5. 再次点击开关（关→开）
6. 验证：状态变回 **"活跃"**

#### **测试 5：删除配置**
1. 找到一个测试配置
2. 点击 **"删除"** 按钮
3. 在确认弹窗中点击 **"确定"**
4. 验证：配置从列表中消失

## 📊 数据统计功能

### **聚类配置统计：**
1. **总配置数** - 系统中所有聚类配置的数量
2. **活跃配置** - 当前激活的配置数量
3. **总运行次数** - 所有配置的累计运行次数
4. **平均质量** - 所有配置的平均轮廓系数

### **引擎执行统计：**
1. **总执行次数** - 三个引擎的总执行次数
2. **规则引擎次数** - 规则引擎单独的执行次数
3. **聚合引擎次数** - 聚合引擎单独的执行次数
4. **关联引擎次数** - 关联引擎单独的执行次数
5. **成功率** - 成功执行次数占比（带进度条）

## ⚠️ 注意事项

### **权限要求：**
- **引擎监控页面**：所有登录用户均可访问
- **聚类配置页面**：仅限 **分析师** 和 **管理员** 角色

### **依赖的后端 API：**
确保后端服务正常运行，以下 API 必须可用：
- `GET /api/v1/clustering` - 获取配置列表
- `POST /api/v1/clustering` - 创建配置
- `PUT /api/v1/clustering/:id` - 更新配置
- `DELETE /api/v1/clustering/:id` - 删除配置
- `POST /api/v1/clustering/:id/activate` - 激活配置
- `POST /api/v1/clustering/:id/deactivate` - 停用配置
- `POST /api/v1/clustering/:id/run` - 运行聚类
- `GET /api/v1/recommendations` - 获取推荐列表
