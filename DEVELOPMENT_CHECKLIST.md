# 开发检查清单

本清单用于确保代码质量和一致性，在提交代码前请逐项检查。

---

## ✅ 后端开发检查清单 (NestJS)

### 代码结构

- [ ] Controller 是否使用了 `@ApiTags` 装饰器
- [ ] 所有接口是否添加了 Swagger 文档（`@ApiOperation`, `@ApiResponse`）
- [ ] 需要认证的接口是否添加了 `@UseGuards(JwtAuthGuard)`
- [ ] 需要权限控制的接口是否添加了 `@Roles()` 装饰器
- [ ] DTO 是否使用了 `class-validator` 装饰器进行验证
- [ ] Entity 的列名是否明确指定（`name: 'column_name'`）
- [ ] Service 方法是否有适当的异常处理
- [ ] 数据库查询是否使用了参数化查询（防止 SQL 注入）

### API 设计

- [ ] RESTful 路径命名是否正确
- [ ] HTTP 方法选择是否恰当（GET/POST/PUT/DELETE）
- [ ] 查询参数是否统一（page, limit, sortBy, order）
- [ ] 响应格式是否符合规范
- [ ] 错误信息是否清晰有用
- [ ] 状态码使用是否正确

### 性能与安全

- [ ] 是否需要添加缓存（Redis）
- [ ] 数据库查询是否需要添加索引
- [ ] 敏感数据是否已加密（密码、token）
- [ ] 输入验证是否完整
- [ ] 是否存在 N+1 查询问题
- [ ] 是否需要添加限流保护

### 测试

- [ ] 是否编写了单元测试
- [ ] 是否编写了 E2E 测试
- [ ] 测试覆盖率是否达标（>80%）
- [ ] 边界情况是否已测试

---

## ✅ 前端开发检查清单 (React)

### 组件开发

- [ ] 是否使用函数组件 + Hooks
- [ ] Props 是否定义了 TypeScript 类型
- [ ] 组件是否足够小（单一职责）
- [ ] 是否避免了不必要的状态
- [ ] useEffect 依赖数组是否正确
- [ ] 是否正确清理了副作用（cleanup）
- [ ] 事件处理器是否正确绑定

### 状态管理

- [ ] 是否合理使用 Zustand store
- [ ] 状态更新是否不可变（immutable）
- [ ] localStorage 使用是否正确
- [ ] 是否在组件卸载时清理状态

### API 调用

- [ ] 是否使用了封装的 apiClient
- [ ] 是否正确处理了 loading 状态
- [ ] 是否正确处理了错误状态
- [ ] Token 是否自动附加到请求头
- [ ] 401 错误处理是否合理

### UI/UX

- [ ] 是否优先使用 Ant Design 组件
- [ ] 样式是否使用 Tailwind CSS
- [ ] 是否支持响应式布局
- [ ] 加载状态是否有友好提示
- [ ] 错误信息是否清晰易懂
- [ ] 表单验证是否完整

### 性能优化

- [ ] 列表数据量大时是否使用虚拟滚动
- [ ] 图片是否使用懒加载
- [ ] 是否避免了不必要的渲染
- [ ] 大组件是否使用 lazy loading
- [ ] 事件监听是否正确移除

### 安全

- [ ] 用户输入是否进行了验证
- [ ] 是否避免了 XSS 攻击风险
- [ ] Token 存储是否安全
- [ ] 敏感信息是否不显示在日志中

## 📝 使用示例

### 开发新功能时

```bash
# 1. 创建分支
git checkout -b feature/new-feature

# 2. 开发过程中对照检查清单
# - 阅读 CODE_STYLE_GUIDE.md
# - 按照规范编写代码
# - 编写测试用例

# 3. 提交前自查
# - 运行 lint: npm run lint
# - 运行测试：npm test
# - 对照检查清单逐项确认

# 4. 提交代码
git add .
git commit -m "feat(module): add new feature"

# 5. 推送到远程
git push origin feature/new-feature

# 6. 创建 Pull Request
# - 填写 PR 描述
# - 指定 Reviewer
# - 等待审核
```

## 🎯 快速参考

### 常用命令

```bash
# 后端
npm run build          # 编译
npm run start          # 启动
npm run test           # 测试
npm run lint           # 检查

# 前端
npm run dev            # 开发模式
npm run build          # 打包
npm run preview        # 预览
npm run lint           # 检查
```

### 快捷方式

- 查看规范文档：`CODE_STYLE_GUIDE.md`
- 查看 API 文档：`http://localhost:3000/api/docs`
- 查看健康状态：`http://localhost:3000/api/v1/health`
