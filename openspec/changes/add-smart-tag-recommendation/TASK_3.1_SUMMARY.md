### 1. 强大的规则表达式

```typescript
// 支持复杂嵌套逻辑
const complexRule = {
  operator: 'AND',
  conditions: [
    { field: 'age', operator: 'between', value: [25, 40] },
    {
      operator: 'OR',
      conditions: [
        { field: 'profile.city', operator: 'in', value: ['北京', '上海'] },
        { field: 'totalAmount', operator: '>=', value: 50000 }
      ]
    },
    { field: 'tags', operator: 'includes', value: 'VIP' }
  ]
};
```

| **数组** | in, includes | `city in ['北京','上海']` |
| **字符串** | startsWith, contains, endsWith | `name startsWith '张'` |

## 🛠️ 技术亮点

### 1. 灵活的表达式语言
- 支持任意深度的嵌套逻辑
- 支持字段路径访问（如 `profile.address.city`）
- 支持多种数据类型和运算符

### 2. 置信度评分机制
- 基于匹配条件比例计算置信度
- AND 表达式：`matchedConditions / totalConditions`
- OR 表达式：`matchedConditions / totalConditions`
- NOT 表达式：反向置信度

### 3. 智能推荐优化
- **自动去重**: 相同标签只保留最高置信度
- **自动排序**: 按置信度降序排列
- **理由生成**: 自动生成人类可读的推荐理由

### 4. 性能优化
- 规则按优先级排序执行
- 惰性加载活跃规则
- 命中次数自动统计

### 遇到的挑战 💪

1. **相对路径导入**
   - NestJS 模块跨目录导入容易出错
   - 解决方案：参考现有文件的导入路径
   - 记忆：从 modules 到 common 需要 `../../common`

2. **TypeORM 装饰器**
   - `@InjectRepository` 需要从正确的包导入
   - `Injectable` 从 `@nestjs/common` 而非 `@nestjs/typeorm`

3. **DTO 验证器**
