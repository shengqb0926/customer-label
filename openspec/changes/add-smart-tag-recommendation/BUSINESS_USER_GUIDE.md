## 3️⃣ 使用规则管理功能

### 3.1 查看规则列表

**访问路径**: 登录后点击左侧菜单的"规则管理"，或直接访问:
```
http://localhost:5173/rules
```

**功能说明**:
- ✅ 查看所有规则（表格形式）
- ✅ 搜索规则（按名称）
- ✅ 筛选规则（按状态：活跃/停用）
- ✅ 分页浏览

**页面元素**:
| 区域 | 说明 |
|------|------|
| **顶部工具栏** | 新建规则、导入规则、刷新列表 |
| **搜索框** | 输入规则名称快速搜索 |
| **状态筛选** | 选择"全部"、"活跃"或"停用" |
| **规则列表** | 显示规则名称、描述、优先级、状态等 |
| **操作列** | 编辑、测试、激活/停用、删除按钮 |

### 3.2 创建新规则

**操作步骤**:
1. 点击规则列表右上角的"➕ 新建规则"按钮
2. 在弹出的表单中填写规则信息:

**必填字段**:
- **规则名称**: 例如"高价值客户识别"
- **规则表达式**: 使用可视化编辑器或 JSON 编辑
- **推荐标签**: 选择或输入要推荐的标签

**可选字段**:
- **描述**: 规则的详细说明
- **优先级**: 1-100，数字越大优先级越高（默认 50）
- **状态**: 活跃/停用

**示例规则**:

#### 示例 1: 高资产客户
```json
{
  "logic": "AND",
  "conditions": [
    {
      "field": "totalAssets",
      "operator": ">=",
      "value": 1000000
    }
  ]
}
```
**含义**: 总资产 >= 100 万的客户

**推荐标签**: `["高净值客户", "VIP 客户"]`

#### 示例 2: 活跃客户
```json
{
  "logic": "AND",
  "conditions": [
    {
      "field": "lastLoginDays",
      "operator": "<=",
      "value": 7
    },
    {
      "field": "orderCount",
      "operator": ">=",
      "value": 5
    }
  ]
}
```
**含义**: 最近 7 天内登录过且订单数 >= 5 的客户

**推荐标签**: `["活跃客户", "高价值客户"]`

#### 示例 3: 流失风险客户
```json
{
  "logic": "OR",
  "conditions": [
    {
      "field": "lastLoginDays",
      "operator": ">",
      "value": 30
    },
    {
      "field": "monthlyIncome",
      "operator": "<",
      "value": 5000
    }
  ]
}
```
**含义**: 超过 30 天未登录 或 月收入 < 5000 的客户

**推荐标签**: `["流失风险客户", "价格敏感客户"]`

3. 点击"保存"按钮

### 3.7 批量导入规则

**操作步骤**:
1. 准备规则 JSON 数据（数组格式）
2. 点击规则列表的"📥 导入"按钮
3. 粘贴 JSON 数据
4. 点击"确认导入"

**导入格式示例**:
```json

  {
    "name": "高净值客户",
    "description": "总资产超过 100 万的客户",
    "expression": {
      "logic": "AND",
      "conditions": [
        {
          "field": "totalAssets",
          "operator": ">=",
          "value": 1000000
        }
      ]
    },
    "tags": ["高净值客户", "VIP 客户"],
    "priority": 80,
    "isActive": true
  },
  {
    "name": "年轻潜力客户",
    "description": "年龄小于 30 岁且收入稳定的客户",
    "expression": {
      "logic": "AND",
      "conditions": [
        {
          "field": "age",
          "operator": "<",
          "value": 30
        },
        {
          "field": "monthlyIncome",
          "operator": ">=",
          "value": 10000
        }
      ]
    },
    "tags": ["潜力客户", "年轻客户"],
    "priority": 60,
    "isActive": true
  }
```

## 5️⃣ 常用运算符参考

|--------|------|------|
| `>` | 大于 | `totalAssets > 1000000` |

| `<=` | 小于等于 | `lastLoginDays <= 7` |

## 7️⃣ 常见问题解答

### Q1: 如何创建复杂的 AND/OR 组合规则？

**答**: 使用嵌套的表达式结构:

```json
{
  "logic": "AND",
  "conditions": [
    {
      "field": "totalAssets",
      "operator": ">=",
      "value": 500000
    },
    {
      "logic": "OR",
      "conditions": [
        {
          "field": "monthlyIncome",
          "operator": ">=",
          "value": 30000
        },
        {
          "field": "annualSpend",
          "operator": ">=",
          "value": 100000
        }
      ]
    }
  ]
}
```

**含义**: 总资产 >= 50 万，且 (月收入 >= 3 万 或 年消费 >= 10 万)

### Q2: 规则优先级如何使用？

**答**: 
- 优先级范围：1-100，数字越大优先级越高
- 当多个规则都匹配时，优先展示高优先级规则的推荐
- 建议设置：
  - 核心业务规则：80-100
  - 一般规则：50-79
  - 辅助规则：1-49

### Q3: 如何调试规则不生效的问题？

**答**: 
1. 检查规则状态是否为"活跃"
2. 使用规则测试工具验证表达式
3. 检查客户数据是否满足条件
4. 查看置信度是否 >= 0.6（阈值）
5. 检查是否有更高优先级的规则冲突

### Q4: 置信度是如何计算的？

**答**: 
- **单个条件**: 置信度 = 1.0（满足）或 0.0（不满足）
- **AND 逻辑**: 置信度 = 所有条件置信度的乘积
- **OR 逻辑**: 置信度 = 1 - (所有条件不置信度的乘积)
- 只有置信度 >= 0.6 才会生成推荐

### Q5: 可以创建多少个规则？

**答**: 
- 理论上无限制
- 建议保持 20-50 个高质量规则
- 过多规则会影响性能

## 8️⃣ 最佳实践建议

### 规则命名规范
✅ **推荐**:
- "高净值客户识别"
- "流失风险预警 -30 天未登录"
- "年轻潜力客户 - 年龄<30"

❌ **不推荐**:
- "规则 1"
- "test"
- "aaa"

### 规则设计原则

1. **单一职责**: 每个规则只关注一个业务场景
2. **明确表达**: 使用清晰的字段和运算符
3. **合理优先级**: 根据业务重要性设置
4. **定期优化**: 根据效果调整规则参数

### 标签推荐策略

1. **精准标签**: 针对特定客户群体（如"高净值客户"）
2. **通用标签**: 覆盖广泛特征（如"活跃客户"）
3. **分层标签**: 结合多个维度（如"高价值流失风险客户"）
